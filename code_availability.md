# Code Availability Section

Here is the structured code availability section for your paper submission based on the analysis of your repository:

* **Name of the code/library:** Dual-Phase Framework for LSM
* **Hardware requirements:** Standard PC with internet access for Google Earth Engine. For deep learning (BiLSTM) and data processing tasks, a GPU (e.g., NVIDIA T4 via Google Colab or local equivalent) and at least 16GB of RAM are recommended.
* **Program language:** Python 3.x, JavaScript (for Google Earth Engine)
* **Software required:** Google Earth Engine (GEE) account, QGIS (v3.x or higher), Jupyter Notebook / Google Colab environment. Required Python packages include `tensorflow` (or `torch`), `scikit-learn`, `rasterio`, `netCDF4`, `pandas`, `numpy`, `xarray`, and others (listed in `requirements.txt`).
* **Program size:** ~17.5 MB (scripts and notebooks only), ~195 MB (entire repository including processed spatial datasets).
* **The source codes are available for downloading at the link:** [https://github.com/283swati/Wayanad-LSM-Zenodo] (Permanently archived via Zenodo: https://doi.org/10.5281/zenodo.20205386)
